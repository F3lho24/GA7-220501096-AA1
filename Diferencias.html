🚀 Git: Versionamiento Local vs Remoto

Entiende las diferencias clave y cómo interactúan

🔧 Versionamiento Local

  • Se ejecuta en tu máquina (sin necesidad de internet).
  • Comandos principales: git init, git add, git commit.
  • Ventajas:
    • Rápido (no depende de servidores externos). ⚡
    • Trabajo offline posible. 📴
    • Control total de tu historial. 🛠️
  • Limitaciones:
    • No permite colaboración. 👥❌
    • Riesgo de perder datos si falla tu disco duro. 💾⚠️

☁️ Versionamiento Remoto

  • Almacena el repositorio en servidores Externos (GitHub, GitLab, Bitbucket).
  • Comandos principales: git clone, git push, git pull.
  • Ventajas:
    • Colaboración en equipo. 👥✅
    • Backup automático del código. 🗄️
    • Integración con CI/CD. 🔄
  • Limitaciones:
    • Requiere conexión a internet. 🌐❌
    • Dependencia del servicio externo. ⏳

🔗 Flujo de Trabajo Combinado

1

Cambios locales (git add + git commit)

→
2

Subir a remoto (git push)

→
3

Colaboradores actualizan (git pull)

✨ Git sincroniza ambos entornos para mantener un historial consistente.
